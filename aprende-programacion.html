Tema 1 · Programación Orientada a Objetos

¿Qué es la POO?

Programar en objetos es como diseñar con moldes. Primero defines el molde (la Clase), luego lo usas para crear cosas reales (Objetos).

📐 Clase (el molde)

Auto

marca: String color: String velocidad: Int acelerar() frenar()
⬇️

Haz clic para crear un objeto (instancia) de la clase Auto:

👆 Haz clic en un botón para crear tu primer objeto
📐
Clase
El molde o plano. Define cómo será algo, pero no es la cosa en sí. "La clase Auto define que todo auto tiene marca, color y puede acelerar."
🚗
Objeto
La cosa real creada a partir del molde. Si la clase es el plano, el objeto es la casa construida. "Mi auto rojo específico es un objeto de la clase Auto."
🏭
Constructor
Método especial que se ejecuta al crear el objeto para darle sus valores iniciales. "Como la línea de ensamblaje: cuando creo el objeto, el constructor le pone su marca y color."
🏷️
Atributos
Las características de un objeto. "En la clase Auto, los atributos serían marca, color y velocidad."
⚡
Métodos
Las acciones que un objeto puede hacer. "acelerar() o frenar() serían métodos de la clase Auto. Son funciones dentro de la clase."
🎯
Instancia
Cada objeto creado a partir de una clase es una instancia. "val miAuto = Auto("Toyota","Rojo") crea una instancia de Auto."

Así se ve en Kotlin 🇰🇷

Kotlin · Android Studio
// 1. La CLASE (el molde) class Auto(marca: String, color: String) { // ATRIBUTOS var velocidad: Int = 0 // MÉTODOS (acciones) fun acelerar() { velocidad += 10 } fun frenar() { velocidad -= 10 } } // 2. CREAR OBJETOS (instancias del molde) val miAuto = Auto("Toyota", "Rojo") val tuAuto = Auto("Honda", "Azul") // 3. USAR métodos del objeto miAuto.acelerar() // velocidad = 10 miAuto.acelerar() // velocidad = 20
Tema 2 · Los 4 Pilares de la POO

Los 4 pilares que debes conocer

Estos conceptos te los pueden preguntar directo en una entrevista. Aquí los ves en acción, no solo de memoria.

🔒 Encapsulamiento

Esconder los datos internos

Se usa private para proteger datos y métodos get()/set() para acceder controladamente.

class CuentaBancaria { private var saldo: Double = 0.0 // ← nadie accede directo fun depositar(monto: Double) { if (monto > 0) saldo += monto // acceso controlado } fun getSaldo() = saldo // solo lectura }

🌳 Herencia

Reutilizar código con una clase padre

La clase hija hereda todo de la clase padre. No repites código.

open class Vehiculo { // PADRE var ruedas: Int = 4 fun arrancar() { /* encender */ } } class Auto : Vehiculo() { // HIJO fun abrirMaletero() { } // propio } class Moto : Vehiculo() { // HIJO init { ruedas = 2 } fun hacerCaballito() { } }

🌳 Árbol de herencia

🚗 Vehiculo
ruedas · arrancar()
🚗 Auto
✓ ruedas · ✓ arrancar()
+ abrirMaletero()
🏍 Moto
✓ ruedas=2 · ✓ arrancar()
+ hacerCaballito()
💡 open en Kotlin significa "esta clase puede ser heredada". Por defecto las clases son cerradas.
🎭 Polimorfismo

El mismo método, comportamientos distintos

El método se llama igual en todos, pero cada clase lo implementa a su manera con override.

open class Animal { open fun hacerSonido(): String = "" } class Perro : Animal() { override fun hacerSonido() = "¡Guau guau!" } class Gato : Animal() { override fun hacerSonido() = "¡Miauu!" } class Pato : Animal() { override fun hacerSonido() = "¡Cuac cuac!" }

🎭 Abstracción

Mostrar lo esencial, ocultar lo complejo

Manejas el auto sin saber cómo funciona el motor por dentro. Las clases abstract obligan a implementar los métodos esenciales.

abstract class Vehiculo { // Obliga a cada hijo a implementarlo abstract fun moverse() // Lo complejo está oculto aquí adentro private fun calcularCombustion() { } } class Auto : Vehiculo() { override fun moverse() { // Solo dices qué hace, no el cómo interno println("Girando ruedas...") } }

Tema 3 · Bases de Datos

¿Cómo se guardan los datos?

Todo app guarda información en algún lugar. Las bases de datos son esos lugares, y cada una tiene sus reglas.

📊 BD Relacional
Base de datos relacional
Como Excel pero con tablas conectadas entre sí. Una tabla de clientes, otra de pedidos, vinculadas. Ejemplos: MySQL, PostgreSQL, SQLite.
🔑 Primary Key
Clave Primaria (PK)
El campo único que identifica cada fila. Como tu DNI: nadie tiene dos iguales. Se escribe PRIMARY KEY.
🔗 Foreign Key
Clave Foránea (FK)
Un campo que conecta dos tablas. En la tabla Pedidos, el id_cliente es FK que apunta a la tabla Clientes.
🔗 JOIN
JOIN
Combina datos de dos tablas en una consulta. "Quiero ver el nombre del cliente junto a su pedido." → JOIN entre ambas tablas.
✅ CRUD
CRUD
Las 4 operaciones básicas: Create (crear), Read (leer), Update (actualizar), Delete (borrar). Todo app de gestión hace CRUD.
🗂 Normalización
Normalización
Organizar las tablas para evitar datos repetidos. Separar info en tablas lógicas: no escribas el nombre del cliente en cada pedido, solo su id.

SQL vs NoSQL — ¿Cuál usar?

🗂 SQL (Relacional)
✅ Tablas con filas y columnas
✅ Relaciones entre tablas
✅ Ideal para: bancos, e-commerce, ERPs
📦 MySQL · PostgreSQL · SQLite (Android)
👉 Úsalo cuando la estructura es fija y necesitas relaciones
📄 NoSQL (No relacional)
✅ Guarda como documentos JSON
✅ Muy flexible, sin esquema fijo
✅ Ideal para: apps sociales, chats, logs
📦 Firebase · MongoDB · Room (Android)
👉 Úsalo cuando los datos cambian mucho o son muy variados
💡 En Android Studio: para apps locales usas Room (que es SQLite con Kotlin). Para la nube, Firebase Firestore (NoSQL) es el más popular.
